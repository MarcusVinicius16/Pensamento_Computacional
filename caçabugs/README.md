# Tratamento de Erros e Avaliação de Soluções – UrbanFlow

Este repositório contém a documentação e as implementações referentes à aula prática de **Tratamento de Erros e Avaliação de Soluções**. O objetivo desta etapa é garantir a robustez, segurança e eficiência do sistema **UrbanFlow**, focando na identificação e correção de falhas de software.

## 🚀 Sobre o Projeto
O **UrbanFlow** é um sistema de monitoramento logístico em tempo real, desenvolvido para otimizar o fluxo de mercadorias em ambientes urbanos. Por ser um sistema que lida com dados críticos de geolocalização e prazos, o tratamento rigoroso de erros é fundamental para evitar falhas de execução e inconsistências no banco de dados.

## 🎯 Objetivos da Atividade
1.  **Identificação:** Mapear erros de sintaxe, lógica e execução (runtime) no código atual.
2.  **Correção e Justificativa:** Aplicar as correções necessárias (ex: blocos `try-catch`, validação de inputs, tratamento de conexões SQL) e documentar o porquê de cada alteração.
3.  **Avaliação Crítica:** Analisar se a solução corrigida é clara para outros desenvolvedores, se é eficiente em termos de processamento e se está pronta para crescer (escalabilidade).

## 📁 Estrutura da Entrega
A documentação detalhada desta etapa foi dividida nos seguintes arquivos:

*   **Erros_Identificados.md**: Registro técnico detalhado de cada falha encontrada antes da correção.
*   **Projeto_Corrigido.md**: Explicação das mudanças aplicadas no código-fonte com suas respectivas justificativas.
*   **Avaliacao.md**: Reflexão sobre os ganhos de qualidade, performance e manutenibilidade após as correções.

## 🛠 Tecnologias Utilizadas
*   PHP (Back-end)
*   MySQL (Banco de Dados)
*   HTML/CSS & JavaScript (Interface)
*   Git/GitHub (Versionamento)
